# Прошивка и настройка GRBL-Mega для Arduino Mega + RAMPS 1.6+

Этот документ описывает процесс установки и первоначальной настройки прошивки **GRBL-Mega-5X** на плату Arduino Mega 2560 с шилдом BigTreeTech RAMPS 1.6+.

## 1. Необходимое ПО

1.  **[Arduino IDE](https://www.arduino.cc/en/software)** (версия 1.8.x или 2.x) — среда для компиляции и загрузки прошивки.
2.  **[GRBL-Mega-5X](https://github.com/fra589/grbl-Mega-5X)** — форк прошивки GRBL, адаптированный для Arduino Mega и поддерживающий до 5 осей. Скачайте его как ZIP-архив (`Code` -> `Download ZIP`).

## 2. Установка библиотеки GRBL (Ручной способ)

Автоматическая установка через "Добавить .ZIP библиотеку..." может вызывать ошибку из-за структуры архива с GitHub. Ручной способ надежнее:

1.  **Распакуйте** скачанный ZIP-архив `grbl-Mega-5X-main.zip`. Вы получите папку с названием `grbl-Mega-5X-main`.
2.  Внутри этой папки находится еще одна папка `grbl-Mega-5X`. **Это и есть папка с библиотекой.**
3.  Найдите папку с библиотеками Arduino на вашем компьютере. Обычно она находится по пути: `C:\Users\<Ваше_Имя>\Documents\Arduino\libraries`.
4.  **Скопируйте** или **переместите** папку `grbl-Mega-5X` (из шага 2) в вашу папку `libraries`.
5.  **Перезапустите Arduino IDE.**
6.  После перезапуска вы сможете найти примеры прошивки в меню `Файл` > `Примеры` > `grbl-Mega-5X`.

## 3. Конфигурация для RAMPS 1.6+

Это самый важный шаг. Нам нужно указать прошивке, что мы используем плату RAMPS, чтобы она правильно назначила выводы (пины) для моторов, концевиков и лазера. Этот файл нужно редактировать **после** установки библиотеки и **до** прошивки платы.

1.  Найдите папку с библиотеками Arduino. Обычно она находится в `C:\Users\<Ваше_Имя>\Documents\Arduino\libraries`.
2.  Перейдите в папку с установленной прошивкой: `grbl-Mega-5X\grbl`.
3.  Откройте файл `config.h` в любом текстовом редакторе.
4.  Найдите раздел, отвечающий за выбор платы. Он выглядит примерно так:
    ```c
    // To use with RAMPS 1.4 boards, uncomment this line.
    // #define DEFAULTS_RAMPS_1_4
    ```
5.  **Раскомментируйте** строку для RAMPS 1.4. Плата RAMPS 1.6+ имеет ту же распиновку для базовых функций, что и 1.4.
    ```c
    // To use with RAMPS 1.4 boards, uncomment this line.
    #define DEFAULTS_RAMPS_1_4
    ```
6.  Найдите и **раскомментируйте** строку для включения режима лазера. Это позволит использовать команды `M3`/`M4`/`M5` и управление мощностью через `S`.
    ```c
    // Enables laser mode.
    // #define LASER_MODE_ENABLE
    ```
    Должно стать:
    ```c
    #define LASER_MODE_ENABLE
    ```
7.  Сохраните и закройте файл `config.h`.

## 4. Прошивка Arduino Mega

1.  Подключите Arduino Mega 2560 к компьютеру.
2.  В Arduino IDE откройте пример `grblUpload`: `Файл` > `Примеры` > `grbl-Mega-5X` > `grblUpload`.
3.  В меню `Инструменты` выберите:
    *   **Плата:** `Arduino Mega or Mega 2560`
    *   **Процессор:** `ATmega2560 (Mega 2560)`
    *   **Порт:** Выберите COM-порт, к которому подключена ваша плата.
4.  Нажмите кнопку `Загрузить` (стрелка вправо).
5.  Дождитесь окончания процесса загрузки.

## 5. Проверка

1.  После успешной прошивки откройте `Инструменты` > `Монитор порта`.
2.  В правом нижнем углу окна Монитора порта установите скорость **115200 бод**.
3.  Вы должны увидеть приветственное сообщение, например: `Grbl-Mega-5X 1.2h ['$' for help]`.
4.  Отправьте команду `$$` в поле ввода и нажмите Enter. В ответ вы должны получить список всех настроек GRBL.

Если вы видите список настроек, значит, прошивка прошла успешно. Теперь можно переходить к подключению драйверов и моторов.

## 6. Устранение неполадок (Troubleshooting)

### Проблема: Пустой экран в Мониторе порта

Вы установили скорость 115200 бод, но в окне Монитора порта ничего не появляется.

*   **Решение 1: Нажмите Reset.** Не закрывая Монитор порта, нажмите физическую кнопку сброса (Reset) на самой плате Arduino Mega. При перезагрузке плата снова отправит приветственное сообщение. Часто это помогает.

*   **Решение 2: Используйте внешнюю программу.** Иногда Монитор порта в Arduino IDE работает нестабильно. Попробуйте подключиться к плате через другую программу, например, **Universal Gcode Sender (UGS)** или LightBurn. Выберите ваш COM-порт, скорость 115200 и нажмите "Подключиться". Консоль в этих программах часто работает надежнее.

*   **Решение 3: Перепрошейте плату.** Попробуйте загрузить скетч `grblUpload` еще раз. Внимательно следите за сообщениями в нижней части окна Arduino IDE. Возможно, при прошлой загрузке произошла ошибка, которую вы не заметили.

*   **Решение 4: Проверьте "железо" (тест с Blink).** Если ничего не помогает, нужно убедиться, что сама плата жива.
    1.  Отключите шилд RAMPS от Arduino Mega.
    2.  Загрузите на плату стандартный пример `Blink` (меню `Файл` > `Примеры` > `01.Basics` > `Blink`).
    3.  Если встроенный светодиод (с маркировкой "L") на плате начал мигать, значит плата, кабель и USB-порт в порядке. Проблема, скорее всего, в самой прошивке GRBL или конфликте с шилдом. Если светодиод не мигает — проблема в самой плате или подключении.
