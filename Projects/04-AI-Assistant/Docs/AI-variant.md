# ⚖️ AI-variant: Выбор стека под RTX 3060

> **Контекст**: Выбор оптимальных моделей для локального запуска.
> **Железо**: GPU RTX 3060 (12GB VRAM), RAM 64 GB, CPU Ryzen 7 5700G.

## 📊 Таблица совместимости (GGUF / Quantization)

| Модель | Параметры | Размер (Q4_K_M) | VRAM (слои на GPU) | Скорость (est.) | Вердикт |
|--------|-----------|-----------------|--------------------|-----------------|---------|
| **Llama-3.1** | 8B | ~4.9 GB | 100% (все слои) | ⚡ Высокая | ✅ Основная (Chat) |
| **Qwen2.5-Coder** | 14B | ~9.0 GB | 100% (впритык) | 🟢 Средняя | ✅ Кодинг |
| **Mistral-Nemo** | 12B | ~7.5 GB | 100% | ⚡ Высокая | 🆗 Альтернатива |
| **Gemma-2** | 27B | ~16-17 GB | ~60-70% (остальное в RAM) | 🐢 Низкая | ❌ Тяжело для 3060 |
| **DeepSeek-R1** | Distill 14B | ~9.0 GB | 100% | 🟢 Средняя | ✅ Логика/RAG |

## 🏆 Рекомендованный стек

### 1. Быстрый чат и рутина
- **Модель**: `llama3.1:8b-instruct-q6_K`
- **Почему**: Полностью влезает в 12GB VRAM с запасом под контекст 8k. Максимальная скорость отклика.

### 2. Кодинг (VS Code / Continue)
- **Модель**: `qwen2.5-coder:14b-instruct-q4_K_M`
- **Почему**: Лучшая модель для кода в этом весе. Влезает в 12GB, если не забивать контекст до предела.
- **Нюанс**: Если нужно держать модель загруженной параллельно с браузером и Houdini, лучше спуститься до версии `7b`.

### 3. Сложные задачи / Анализ (Slow mode)
- **Модель**: `claude-3-5-sonnet` (через API) — для архитектуры и сложных рефакторингов.
- **Локально**: `gemma-2:27b-instruct-q4_K_M` (с выгрузкой части слоев в 64GB RAM). Будет медленно (2-5 t/s), но качественно для локального анализа больших текстов.

## ⚙️ Настройки Ollama (Modelfile)

Для моделей 12-14B на карте с 12GB памяти критически важно контролировать контекст, чтобы избежать вылетов (OOM) или резкого падения скорости при переполнении VRAM.

Пример для Qwen 14B:
```dockerfile
FROM qwen2.5-coder:14b
PARAMETER num_ctx 8192
PARAMETER num_gpu 999
```
