## 🏭 Часть 1: Houdini + CNC / Лазер

### Зачем это нужно?
G-code — это просто список координат и команд. Houdini умеет:
- **визуализировать траекторию** до запуска станка,
- находить **коллизии** и нештатные движения,
- **симулировать** процесс резки в реальном времени.

### Как устроен G-code (упрощённо)

```
G0 X0 Y0 Z5       ; быстрое перемещение в начало
G1 X10 Y0 Z0 F300 ; линейное движение со скоростью 300
G1 X10 Y10 Z0
G1 X0  Y10 Z0
G1 X0  Y0  Z0
```

Для визуализации нам нужны только **X Y Z** координаты.

### Схема SOP-сети для визуализации G-code

```
[File / Python SOP]         ← читаем координаты
       ↓
[Add]                       ← строим ломаную линию
       ↓
[Resample]                  ← увеличиваем плотность точек
       ↓
[Copy to Points] ← [Sphere] ← копируем "головку" инструмента
       ↓
[OUT_toolpath]              ← Null-нода для экспорта
```

### Ключевые ноды

| Нода | Назначение |
|------|-----------|
| **File SOP** | Чтение внешних данных (`.txt`, `.csv`) |
| **Python SOP** | Парсинг G-code, нестандартные форматы |
| **Add** | Создание линий между точками |
| **Resample** | Равномерное распределение точек по кривой |
| **Carve** | Анимация движения по кривой (параметр First U: 0→1) |
| **Copy to Points** | Копирование геометрии (инструмента) на точки пути |
| **Trail** | Визуализация "следа" за инструментом |

### VEX: цвет траектории по высоте (Z)

```c
// Attribute Wrangle (Point) — раскраска пути по Z
float zmin = getbbox_min(0).z;
float zmax = getbbox_max(0).z;
@Cd = set(fit(@P.z, zmin, zmax, 0.0, 1.0), 0.3, 0.8);
```

## 🖨️ Часть 2: Houdini + 3D-печать

### Зачем это нужно?
Перед печатью важно знать:
- есть ли **нависающие элементы** (overhang > 45°)?
- нужны ли **поддержки**?
- корректны ли **нормали** модели?

Houdini позволяет **автоматически найти проблемные зоны** за несколько нод.

### Анализ углов нависания (Overhang Analysis)

**Принцип:** угол между нормалью грани и вектором "вниз" (0, -1, 0).
Если угол > 45° — нависание критическое.

```c
// Attribute Wrangle (Primitive) — анализ нависания
vector down = {0, -1, 0};
float angle = degrees(acos(dot(normalize(@N), down)));

// Раскраска: зелёный = ок, красный = нависание
if (angle < 45) {
    @Cd = set(0.2, 0.8, 0.2);  // зелёный — безопасно
} else if (angle < 60) {
    @Cd = set(1.0, 0.6, 0.0);  // оранжевый — пограничная зона
} else {
    @Cd = set(0.9, 0.1, 0.1);  // красный — нужна поддержка
}
```

### Схема SOP-сети для анализа модели

```
[File SOP / Import]         ← загружаем .STL / .OBJ
       ↓
[Normal SOP]                ← пересчитываем нормали
       ↓
[Attribute Wrangle]         ← VEX: анализ угла нависания
       ↓
[Blast]                     ← (опц.) изолируем проблемные полигоны
       ↓
[OUT_overhang_analysis]
```

### Ключевые ноды для работы с моделями

| Нода | Назначение |
|------|-----------|
| **File SOP** | Импорт `.stl`, `.obj`, `.fbx` |
| **Normal SOP** | Пересчёт нормалей (важно для корректного анализа) |
| **PolyDoctor** | Поиск и исправление ошибок меша |
| **Measure** | Измерение площади, объёма, периметра |
| **Blast** | Удаление / изоляция геометрии по атрибуту |
| **Bound** | Bounding box — проверка габаритов |

### Проверка габаритов (влезет ли в принтер?)

```c
// Detail Wrangle — проверка размеров модели
vector size = getbbox_size(0);
printf("Размер модели: X=%.1f Y=%.1f Z=%.1f мм\n", size.x, size.y, size.z);

// Максимальный стол: 220x220x250 мм (пример для Ender 3)
if (size.x > 220 || size.y > 220 || size.z > 250) {
    printf("⚠️ МОДЕЛЬ НЕ ВЛЕЗАЕТ В ПРИНТЕР!\n");
}
```
